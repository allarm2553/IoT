# ทักษะ: การสร้างและตั้งค่า ESP32-S3 Tailscale VPN + MQTT Sensor Gateway

เอกสารทักษะนี้มีวัตถุประสงค์เพื่อให้คำแนะนำและขั้นตอนการทำงานทั้งหมดในการคอมไพล์ ตั้งค่า และติดตั้งอุปกรณ์โครงข่ายร่วม (Hybrid) ESP-IDF/Arduino Tailscale VPN Node ที่เชื่อมเข้ากับระบบควบคุม PicoMQTT Broker และหน้าเว็บควบคุมสไตล์ Glassmorphic ท้องถิ่นบนบอร์ดพัฒนา **ESP32-S3 DevKitC-1**

---

## 🛠️ 1. พินพอร์ตฮาร์ดแวร์และการเชื่อมต่ออุปกรณ์ต่อพ่วง (Pinout Mapping)

| อุปกรณ์ต่อพ่วง | รายละเอียดฮาร์ดแวร์ / คุณลักษณะ | พิน GPIO บนบอร์ด |
| :--- | :--- | :--- |
| **ไฟแสดงสถานะ RGB LED** | ไฟสถานะออนบอร์ด WS2812B Addressable RGB LED | **GPIO 48** (ควบคุมผ่านฟังก์ชัน `neopixelWrite`) |
| **ปุ่มรีเซ็ตระบบ** | ปุ่ม BOOT ออนบอร์ด (สัญญาณแบบ Active-Low) | **GPIO 0** (ตรวจสอบการกดค้าง 3 วินาทีในลูปทำงานหลัก) |
| **หน้าจอแสดงผล OLED** | หน้าจอ I2C SSD1306 ขนาด 128x64 พิกเซล | **SDA = GPIO 8**, **SCL = GPIO 9** (ระบุพารามิเตอร์ผ่าน `Wire.begin()`) |

### ระบบรหัสสีของไฟสถานะ RGB LED
*   **ไฟกะพริบสีน้ำเงิน (ทุก 500ms)**: บอร์ดรันอยู่ในโหมด Access Point สำหรับหน้าเว็บตั้งค่าเครือข่าย (AP Portal Mode)
*   **ไฟกะพริบสีส้ม/เหลือง (ทุก 500ms)**: กำลังพยายามเชื่อมต่อและเชื่อมโยงคลื่นสัญญาณ WiFi STA
*   **ไฟสีเขียวค้าง (Solid Green)**: เชื่อมต่อ WiFi สำเร็จแล้ว แต่ VPN ของ Tailscale ยังไม่ทำงานหรืออยู่ในระหว่างเชื่อมต่อ
*   **ไฟสีฟ้าค้าง (Solid Cyan)**: เชื่อมต่อ WiFi สำเร็จ และอุโมงค์เชื่อม VPN ของ Tailscale ออนไลน์พร้อมรับส่งข้อมูลแล้ว
*   **ไฟกะพริบสีส้ม/แดงอย่างรวดเร็ว (ทุก 100ms)**: สัญญาณเตือนขณะกำลังกดปุ่ม BOOT ค้าง
*   **ไฟกะพริบสีแดงรวดเร็ว (500ms เป็นเวลา 2s)**: ตอบรับการรีเซ็ตบอร์ดกลับค่าโรงงาน และกำลังเคลียร์ข้อมูล NVS / คีย์ข้อมูลเดิม

## 📂 2. โครงสร้างโฟลเดอร์โครงการและไฟล์สำคัญ

```text

    └── main.cpp                  # ไฟล์เฟิร์มแวร์หลักที่รวบรวมฟังก์ชันทำงานทั้งหมด
```

## ⚙️ 3. การกำหนดค่าที่สำคัญสำหรับการคอมไพล์ (Compilation Configurations)

### 1. การตั้งค่าใน platformio.ini
ตรวจสอบให้แน่ใจว่าได้ระบุไลบรารีและเวอร์ชันที่สอดคล้องกับพารามิเตอร์ `lib_deps` เพื่อป้องกันปัญหาระบบหาฟังก์ชันไม่เจอตอน Linkage:
```ini
lib_deps =
    LittleFS
    FS
    PicoMQTT
    Adafruit GFX Library
    Adafruit SSD1306
```

### 2. การกำหนดค่าใน sdkconfig.defaults
ค่าเหล่านี้มีความ**จำเป็นอย่างยิ่ง**เพื่อป้องกันความจำล้น (Stack Overflow) และปัญหาการรีบูตของระบบ:
```ini
# กำหนดให้เริ่มฟังก์ชัน setup() และ loop() ของ Arduino บนระบบ ESP-IDF
CONFIG_AUTOSTART_ARDUINO=y

# ตั้งค่าความถี่ RTOS Tick Rate เป็น 1000Hz (จำเป็นสำหรับ Arduino Core)
CONFIG_FREERTOS_HZ=1000

# เปิดและตั้งค่าการทำงาน ESP32-S3 Octal PSRAM (สำหรับบอร์ดตระกูล N8R8)
CONFIG_SPIRAM_MODE_OCT=y
CONFIG_SPIRAM_MODE_QUAD=n
CONFIG_SPIRAM_SPEED_80M=y
CONFIG_SPIRAM_TYPE_AUTO=y
CONFIG_SPIRAM_SUPPORT=y

# กำหนดขนาดพื้นที่ Stack สำหรับ Arduino Loop task
CONFIG_ARDUINO_LOOP_STACK_SIZE=32768
```

### 3. การจัดขนาดหน่วยความจำของงาน (ในไฟล์ microlink_internal.h)
การคำนวณด้านวิทยาการเข้ารหัสลับของ Tailscale (การทำงานของโปรโตคอล WireGuard และ handshakes ผ่าน mbedTLS) ต้องใช้พื้นที่ Stack เพิ่มขึ้น:
*   `ML_TASK_NET_IO_STACK`: **16 KB**
*   `ML_TASK_DERP_TX_STACK`: **32 KB**
*   `ML_TASK_COORD_STACK`: **24 KB**
*   `ML_TASK_WG_MGR_STACK`: **16 KB**

## 🔄 4. การจัดการสถานะของระบบและการตรวจสอบปัญหา (Troubleshooting)

### 1. การล้างและสลับคีย์ของ Tailscale (Clean Registration)
ตัวระบบ MicroLink จะแคชข้อมูลคีย์ส่วนตัวของ WireGuard (Private Keys) และรายชื่อบอร์ดปลายทางไว้ในหน่วยความจำ NVS บนบอร์ดผ่านเนมสเปซ `"microlink"` ดังนั้นหากมีการส่งคีย์เชื่อมต่อ Tailscale Auth Key ตัวใหม่เข้ามาจากพอร์ตตั้งค่า บอร์ด ESP32-S3 จะต้องรีเซ็ตและล้างค่าตัวตนเดิมทิ้งเพื่อไม่ให้เชื่อมต่อซ้อนกัน:
*   **เงื่อนไข**: เปรียบเทียบค่า `tailscale_auth_key` ตัวใหม่กับ `registered_auth_key` ตัวเก่าที่บันทึกอยู่ใน `/config.txt`
*   **การแก้ไข**: บอร์ดจะสั่งใช้ `microlink_factory_reset()` เพื่อลบเนมสเปซ NVS เดิม แล้วทำการบันทึกคีย์ใหม่เป็น `registered_auth_key` ก่อนจะรีบูตระบบใหม่

### 2. การสลับโหมดการทำงานระหว่าง STA และ AP_STA Mode
การพยายามสลับโหมดควบคุม WiFi ขณะที่บอร์ดกำลังเชื่อมต่อสัญญาณคลื่นอาจจะทำให้บอร์ดเกิดปัญหาตื่นตระหนก (Kernel Panic) จากภาวะแย่งชิงทรัพยากร (Race Conditions)
*   **การแก้ไข**: ต้องตัดการเชื่อมต่อและปิดระบบควบคุม WiFi ก่อนการจัดโหมดการสลับพอร์ตเสมอ:
    ```cpp
    WiFi.disconnect(true);
    delay(100);
    WiFi.mode(WIFI_OFF);
    delay(200);
    WiFi.mode(WIFI_AP_STA);
    delay(200);
    WiFi.softAP(apName.c_str(), "");
    ```

### 3. การตรวจสอบเส้นทางหน่วยเครือข่ายจำลอง (Virtual Network Interface Routing)
เนื่องจากพอร์ตอินเตอร์เฟสของ WireGuard จะเชื่อมต่อข้ามมาตรฐานเลเยอร์ `esp_netif_t` โดยตรงไปยังระบบจำลองเน็ตเวิร์ก lwIP's `netif_list` ดังนั้น ตัว WebServer จะต้องถูกสร้างมาให้รับคำขอจากพอร์ตไอพีใดก็ได้ (`INADDR_ANY` หรือ `0.0.0.0`) เพื่อตอบสนองต่อผู้ใช้งานจากภายนอกและภายในได้อย่างสอดคล้อง ดังนั้นให้ใช้คำสั่ง `WebServer server(80)` โดยไม่ระบุผูกพอร์ตไอพีเฉพาะ

## 🗑️ 5. ขั้นตอนการรีเซ็ตกลับค่าโรงงาน (Factory Reset) ด้วยปุ่มฮาร์ดแวร์

เพื่อลบข้อมูลการตั้งค่าเครือข่าย WiFi และล้างคีย์ความปลอดภัยของ Tailscale VPN ทั้งหมดทางกายภาพ:
1.  ระบุตำแหน่งของ **ปุ่ม BOOT (GPIO 0)** บนแผงวงจร ESP32-S3
2.  **กดปุ่มค้างไว้เป็นเวลา 3 วินาที** ไฟสถานะ RGB LED จะเริ่มกะพริบเตือนสีส้ม/แดงอย่างรวดเร็ว
3.  หลังจากพ้น 3 วินาทีไปแล้ว หน้าจอ OLED จะขึ้นคำเตือนว่า `Factory Resetting` และไฟ LED จะกะพริบสีแดงรวดเร็ว
4.  ระบบจะดำเนินการลบไฟล์ `/config.txt` ออกจากระบบหน่วยความจำ LittleFS และล้างคีย์ WireGuard ในหน่วยความจำ NVS ทั้งหมด ก่อนจะทำการรีบูต
5.  เมื่อบอร์ดเปิดขึ้นมาใหม่ บอร์ดจะสลับเข้าสู่ **โหมดตั้งค่าเครือข่าย (AP Setup Portal Mode)** (ไฟแสดงสถานะ LED กะพริบสีน้ำเงิน) และเริ่มกระจายสัญญาณคลื่น WiFi เฉพาะตัวภายใต้ชื่อ `ESP32-S3-Gateway-[node_id]`
